# Neuro-Symbolic Public Safety Monitor

A lightweight video analytics pipeline for detecting potentially unsafe activity in surveillance footage. The system combines computer vision, deterministic rule-based scoring, and optional large-language-model reasoning to produce an auditable safety assessment.

## What it does

The project processes a video in three stages:

1. Perception
   - Uses YOLOv8 object tracking to detect people and flagged objects such as knives, baseball bats, and scissors.
   - Tracks movement through configured restricted zones.
   - Extracts behavior cues such as crouching, running, and loitering.

2. Symbolic scoring
   - Converts the detected events into a structured severity timeline.
   - Applies deterministic rules to assign a score and severity level to each subject.
   - Produces a transparent, explainable safety report even if the LLM step is unavailable.

3. Causal reasoning (optional)
   - If a high-severity or critical event is detected, the pipeline can invoke a Gemma-based reasoning engine to generate a narrative explanation.
   - If the LLM call fails, the system still falls back to the symbolic report.

## Project structure

- main.py: entry point for the full pipeline
- config.py: tunable thresholds, model names, zones, and severity weights
- perception/: video processing, tracking, and behavior analysis
- reasoning/: symbolic scoring logic and LLM reasoning engine
- tests/: deterministic rule-based tests
- tools/: utilities such as zone calibration
- output/: generated reports for each run

## Installation

1. Create and activate a virtual environment (recommended):

```bash
python -m venv .venv
source .venv/bin/activate
```

2. Install the required Python packages:

```bash
pip install -r requirements.txt
```

3. Ensure Ultralytics YOLOv8 is available:

```bash
pip install ultralytics
```

This project uses Ultralytics YOLOv8. The repository does not include the pretrained YOLO weights file yolov8n.pt, so the first run will download or require that model file when the detector initializes.

The project expects:

- Python 3.10+
- PyTorch-compatible environment
- Optional GPU support for faster inference, though CPU execution is possible

## Usage

Run the pipeline against a video file:

```bash
python main.py --video data/security_footage.mp4 --output-dir output
```

You can also omit the video argument if your default path is configured in config.py.

## Output files

The pipeline writes the following artifacts to the output directory:

- timeline.json: structured event timeline produced by the perception layer
- severity.json: deterministic severity scores and flags
- forensic_report.md: optional narrative explanation generated by the reasoning layer

## Configuration

Most behavior and scoring settings are defined in config.py:

- Video source and frame skip
- Confidence threshold for detections
- Restricted zones
- Threat labels
- Severity weights and thresholds

You can adapt these values to your own environment or camera layout.

## Testing

Run the rule-based tests with:

```bash
pytest -q
```

These tests validate the deterministic scoring behavior without requiring a video or model inference.

## Notes

- The system is intentionally designed to be conservative and auditable.
- Threat detection depends on the quality of the input video and the configured zones.
- The LLM component is optional and is only used when the symbolic scorer finds meaningful severity.
- The pretrained YOLO weights file yolov8n.pt is not bundled in this repository, so you will need Ultralytics/YOLOv8 support available in your environment for inference.
- Local video files placed in the data directory are intended to stay on your machine and are not meant to be pushed to GitHub. When running the pipeline, place your input videos in the data directory so they are picked up correctly.

## Optional tooling

The repository also includes a zone calibration utility in tools/zone_calibrator.py for tuning the restricted-area polygons used by the pipeline.
